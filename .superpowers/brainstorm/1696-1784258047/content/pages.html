美食地图 — 页面结构与路由

移动端 H5 / MVP 五页面

路由设计

GET /

🏠 首页 — 发现美食

核心页面
🗺️ 地图区域 (60%)
高德地图，标记点展示已审核美食
拖动/缩放加载新区域
点击标记 → 跳转详情
📋 列表区域 (40%)
附近美食卡片列表
按距离/时间排序
分类标签快速筛选
与地图区域联动滚动
底部 Tab： 发现 上传 我的
GET /upload

📸 上传美食

1 拍照或选择照片（压缩后上传）
2 填写菜名、选择分类、简短描述
3 自动定位 → 可手动调整 → 确认地址
4 提交 → 提示"等待审核" → 回到首页
GET /food/:id

🍜 美食详情

食物大图 → 标题/分类/描述 → 上传者信息 → 小地图显示位置 → "去这里"导航按钮
GET /admin/review

🛡️ 审核后台

待审核列表（按时间倒序）→ 查看图片/信息 → 通过 ✅ / 拒绝 ❌ + 备注 → 刷新列表
GET /profile

👤 我的

头像/昵称 → 我上传的美食列表（含审核状态）→ 统计：共上传 X 篇，Y 篇通过

路由表一览

GET /首页（地图+列表）
GET /upload上传美食页
GET /food/:id美食详情页
GET /profile个人中心
GET /admin/review审核后台
GET /login微信授权登录
