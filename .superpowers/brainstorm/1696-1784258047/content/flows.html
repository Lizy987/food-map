美食地图 — 核心流程

上传 → 审核 → 展示 / 边界情况处理

流程一：用户上传美食

1
打开上传页 需要登录 → 未登录则跳转微信授权
2
拍照/选择照片 前端压缩至 ≤ 1MB，预览确认
3
填写信息 菜名（必填）+ 分类（下拉选）+ 描述（选填）
4
确认位置 自动获取 GPS → 地图上可拖动微调 → 显示地址文本
5
提交 图片先上传 → 拿 URL → 提交帖子 → 状态 = pending → 提示"审核中"

流程二：审核机制（先审后发）

📥 pending
待审核
→
✅ approved
已通过 → 地图展示
↘
❌ rejected
已拒绝 → 仅上传者可见
审核后台操作：
• 管理员登录 → 查看待审核列表（按时间倒序）
• 查看图片、标题、分类、位置信息
• 点击"通过" → 帖子立即在地图上可见
• 点击"拒绝" → 填写原因（选填）→ 通知上传者
• MVP 阶段：简单 HTTP Basic Auth 保护审核页

边界情况处理

🚫 图片过大

前端压缩到 ≤ 1MB，超过则弹提示"图片过大，请重新选择"

📍 定位失败

提示用户手动在地图上选点，输入地址关键词搜索

🌐 网络异常

上传/提交失败时显示 toast 错误提示，支持重试

🗺️ 地图无数据

当前区域无美食时显示空状态："这片区域还没有人分享美食，来做第一个吧！"

📭 空列表

列表为空时显示插图和引导文字

🔒 未登录保护

上传、审核、个人中心页检测登录态，未登录跳转登录页
