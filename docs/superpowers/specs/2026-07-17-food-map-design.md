# 美食地图 — 设计规格说明

> 日期：2026-07-17 | 状态：已确认

## 1. 产品概述

一个移动端 H5 美食地图应用。用户可以上传食物照片和位置信息，审核通过后在地图上公开展示。以"发现美食"为核心驱动，"分享贡献"为辅助。

### 核心定位

- **目标市场**：中国大陆用户
- **平台形态**：移动端 H5 网页
- **核心动机**：发现为主、分享为辅
- **内容审核**：先审后发
- **发现方式**：地图 + 列表混合布局

### 第一期范围（MVP）

- 上传美食照片 + 位置 + 分类标签
- 地图浏览已审核美食
- 列表浏览附近美食
- 基础审核后台
- 微信授权登录

### 明确不做（YAGNI）

评论、点赞/收藏、用户关注、推送通知、完整管理后台面板。

## 2. 技术方案

**方案 C：混合渐进方案**

| 层 | 选型 | 备注 |
|---|---|---|
| 前端 | React + Vite + TypeScript + Tailwind CSS | 移动端 H5 响应式 |
| 地图 | 高德地图 JS API 2.0 | 国内市场首选 |
| 状态管理 | Zustand | 轻量 |
| 后端 | Bun + Hono + TypeScript | 单进程全栈 |
| 数据库 | SQLite（MVP）→ PostgreSQL（扩展） | 零配置启动 |
| 图片存储 | 本地磁盘（MVP）→ 阿里云 OSS（上线） | 接口抽象，平滑切换 |
| 内容审核 | 人工审核（MVP）→ 接入阿里云/腾讯云 API | 先审后发 |
| 部署 | 单台轻量云服务器 + Docker + Nginx | — |

## 6. 核心流程

### 上传流程

1. 打开上传页（需登录，未登录跳转微信授权）
2. 拍照/选图 → 前端压缩至 ≤ 1MB → 预览
3. 填写菜名（必填）+ 分类（下拉选）+ 描述（选填）
4. 确认位置：自动 GPS → 地图拖动微调 → 地址文本
5. 提交：图片先上传 → 拿 URL → 提交帖子 → status = pending → 提示"审核中"

### 审核状态机

```
pending ──→ approved ──→ 地图公开展示
       └──→ rejected ──→ 仅上传者可见
```

### 审核后台操作

- 管理员查看待审核列表（时间倒序）
- 查看图片、标题、分类、位置
- 通过 → 帖子立即可见
- 拒绝 → 填写原因（选填）
- MVP 阶段：HTTP Basic Auth 保护审核页

## 7. 边界情况处理

| 场景 | 处理 |
|---|---|
| 图片过大 | 前端压缩到 ≤ 1MB，超限提示重新选择 |
| 定位失败 | 引导手动在地图上选点，支持地址搜索 |
| 网络异常 | 上传/提交失败 toast 提示 + 重试 |
| 地图无数据 | 空状态："这片区域还没有人分享美食，来做第一个吧！" |
| 列表为空 | 插图 + 引导文字 |
| 未登录保护 | 上传/审核/个人中心检测登录态，跳转登录页 |

---

## 8. 错误处理

| 层 | 策略 |
|---|---|
| 前端 | API 统一封装，网络错误 toast + 重试；空状态/加载骨架屏 |
| 后端 | Hono 全局错误中间件，统一 `{ error: { code, message } }`；参数校验用 Zod |
| 数据库 | 写入失败返回 500，读异常降级为空数组 |
