# 美食地图 — 设计规格说明

> 日期：2026-07-17 | 状态：已确认 | 版本：v2

## 1. 产品概述

一个**个人使用**的移动端 H5 美食地图。自己上传食物照片、菜名、店名、手动在地图上标记位置，所有记录在地图上可视化展示。没有多用户、没有登录、没有审核——上传即展示。

### 核心定位

- **使用场景**：个人美食记录与地图可视化
- **平台形态**：移动端 H5 网页
- **用户**：仅自己，无需登录注册
- **定位方式**：手动在地图上选点，不用 GPS 自动定位
- **上传流程**：上传即发布，无需审核

### 功能范围

- 上传美食照片 + 菜名 + 店名 + 手动地图选点
- 首页地图展示所有美食标记
- 列表浏览已上传美食
- 美食详情查看

### 明确不做（YAGNI）

多用户、登录注册、审核流程、GPS 自动定位、评论、点赞、收藏、关注、推送。

## 2. 技术方案

| 层 | 选型 | 备注 |
|---|---|---|
| 前端 | React + Vite + TypeScript + Tailwind CSS | 移动端 H5 响应式 |
| 地图 | 高德地图 JS API 2.0 | 国内市场首选 |
| 后端 | Bun + Hono + TypeScript | 单进程全栈 |
| 数据库 | SQLite | 零配置，单文件 |
| 图片存储 | 本地磁盘 | 上传目录 `uploads/` |
| 部署 | 单台轻量云服务器 + Docker + Nginx | — |

---

## 3. 数据模型

单表即可，无需用户表、审核表。

### food_posts — 美食记录

| 字段 | 类型 | 说明 |
|---|---|---|
| id | TEXT (UUID) | 主键 |
| image_url | TEXT | 食物图片路径 |
| dish_name | TEXT | 菜名（必填） |
| store_name | TEXT | 店名（必填） |
| category | TEXT | 分类：火锅、川菜、粤菜、日料、韩餐、西餐、烧烤、小吃、甜品、咖啡、其他 |
| note | TEXT | 备注（选填） |
| latitude | REAL | 纬度（手动选点） |
| longitude | REAL | 经度（手动选点） |
| address | TEXT | 地址文本（高德逆地理编码获取） |
| created_at | TEXT | 上传时间 |

### 索引

- `food_posts(latitude, longitude)` — 地理位置范围查询
- `food_posts(created_at DESC)` — 按时间排序

## 4. 页面与路由

| 路由 | 页面 | 说明 |
|---|---|---|
| `GET /` | 首页 | 地图区域 + 底部列表卡片，地图标记所有美食 |
| `GET /upload` | 上传美食 | 选图 → 填菜名/店名/分类 → 地图上手动点选位置 → 提交 |
| `GET /food/:id` | 美食详情 | 大图 + 菜名/店名 + 小地图显示位置 |

---

## 5. API 端点

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/posts` | 列表查询，参数：lat, lng, radius, page, size |
| GET | `/api/posts/:id` | 单条详情 |
| POST | `/api/posts` | 创建记录（图片+信息），直接入库展示 |
| POST | `/api/upload` | 上传图片，前端压缩至 ≤ 1MB |
| GET | `/api/categories` | 获取分类列表（静态枚举） |

共 5 个端点，无需认证。

---

## 6. 核心流程

### 上传流程

1. 打开上传页
2. 拍照/选图 → 前端压缩至 ≤ 1MB → 预览
3. 填写菜名（必填）、店名（必填）、分类（下拉选）、备注（选填）
4. 地图上手动点击/拖动选点 → 逆地理编码获取地址文本
5. 提交：图片先上传 → 拿 URL → 创建记录 → 直接入库 → 跳转首页

### 首页展示

- 地图区域显示所有美食标记点
- 底部列表按时间倒序展示
- 点击标记/列表项 → 跳转详情页
- 拖动地图 → 列表联动筛选当前视野内的美食

## 7. 边界情况处理

| 场景 | 处理 |
|---|---|
| 图片过大 | 前端压缩到 ≤ 1MB，超限提示 |
| 地图选点未操作 | 提交时校验，提示"请在地图上选择位置" |
| 必填字段为空 | 前端表单校验 + 后端 Zod 校验 |
| 网络异常 | toast 提示 + 重试 |
| 地图无数据 | 空状态提示："还没有美食记录，去上传第一个吧！" |
| 图片上传失败 | 提示重试，不创建记录 |

---

## 8. 错误处理

| 层 | 策略 |
|---|---|
| 前端 | API 统一封装，网络错误 toast + 重试；加载态/空状态/错误态覆盖 |
| 后端 | Hono 全局错误中间件，统一 `{ error: { code, message } }`；参数校验用 Zod |
| 数据库 | 写入失败返回 500，读异常降级为空数组 |
