# 美食地图 — 产品需求文档（PRD）

> 版本：v1.1 | 日期：2026-07-17 | 作者：个人项目

---

## 1. 产品目标与范围

### 1.1 产品定位

一款**个人使用**的移动端 H5 工具。用户拍摄美食照片，手动标记地理位置并填写地址，以地图可视化方式管理个人美食记录。**本地运行，无需网络服务依赖。**

### 1.2 目标用户

- 仅产品所有者本人
- 无多用户体系、无登录、无权限控制

### 1.3 核心价值

将零散的美食照片转化为**空间可视化**的个人美食档案——"我在哪里吃过什么"一目了然。

### 1.4 功能边界

| 范围 | 内容 |
|---|---|
| **本期包含** | 地图浏览美食标记、上传美食（照片+菜名+店名+地址+分类+手动选点）、美食列表、美食详情、编辑记录、删除记录、CSV 导出 |
| **本期不做** | 多用户、登录注册、GPS 自动定位、逆地理编码、审核流程、评论、点赞、收藏、关注、推送通知 |

## 3. 核心业务流程

### 3.1 上传美食流程

```
[拍照/选图] → [前端压缩 ≤ 1MB] → [预览确认]
     ↓
[填写表单] → 菜名(必填) + 店名(必填) + 分类(下拉) + 地址(必填，手动输入) + 备注(选填)
     ↓
[地图选点] → 手动拖动标记到目标位置
     ↓
[提交] → 图片上传至服务端 → 创建记录写入 SQLite → 跳转首页
```

### 3.2 编辑美食流程

```
[详情页点击"编辑"] → 跳转 /food/:id/edit
     ↓
[表单预填] → 图片、菜名、店名、分类、地址、备注、地图标记 均回填
     ↓
[用户修改] → 图片可更换/保留，其他字段可改
     ↓
[提交] → 图片如有更换则上传新图 → 更新 SQLite 记录 → 跳转详情页
```

### 3.3 删除美食流程

```
[详情页点击"删除"] → 弹出确认对话框 → "确定删除这条记录？"
     ↓ 确认
[调用 DELETE API] → 删除数据库记录 + 关联图片文件 → 跳转首页
     ↓ 取消
[关闭对话框，留在详情页]
```

### 3.4 CSV 导出流程

```
[首页点击"导出CSV"] → GET /api/posts/export/csv
     ↓
服务端查询全部记录 → 生成 CSV 文件 → 浏览器触发下载
```

### 3.5 首页浏览流程

```
[打开首页] → 地图加载 → 请求全部美食数据
     ↓
[地图展示] ← 标记点渲染
[底部列表] ← 同批数据，时间倒序
     ↓
[拖动地图 / 选择分类] → 前端侧筛选（数据已在内存）
     ↓
[点击标记/列表项] → 跳转详情页
```

### 3.6 交互联动规则

- 分类筛选 → 地图和列表同时过滤
- 列表项点击 → 地图定位到对应标记并高亮
- 标记点点击 → 弹出气泡显示菜名，点击气泡进详情
- 上滑列表 → 分页加载更多

## 4. 数据模型

### 4.1 表结构：`food_posts`

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | TEXT | PK, UUID | 唯一标识 |
| `dish_name` | TEXT | NOT NULL | 菜名 |
| `store_name` | TEXT | NOT NULL | 店名 |
| `category` | TEXT | NOT NULL | 分类枚举值 |
| `address` | TEXT | NOT NULL | 地址（手动填写） |
| `image_url` | TEXT | NOT NULL | 图片存储路径 |
| `latitude` | REAL | NOT NULL | 纬度（地图手动选点） |
| `longitude` | REAL | NOT NULL | 经度（地图手动选点） |
| `note` | TEXT | — | 备注 |
| `created_at` | TEXT | DEFAULT NOW | 创建时间 |
| `updated_at` | TEXT | DEFAULT NOW | 最后修改时间 |

### 4.2 分类枚举

```
火锅 | 川菜 | 粤菜 | 日料 | 韩餐 | 西餐 | 烧烤 | 小吃 | 甜品 | 咖啡 | 其他
```

### 4.3 索引

- `(latitude, longitude)` — 地理范围查询
- `(created_at DESC)` — 时间排序

## 5. 技术栈

| 层 | 技术选型 | 说明 |
|---|---|---|
| 前端框架 | React 18 + Vite + TypeScript | SPA，移动端适配 |
| UI 样式 | Tailwind CSS | 原子化 CSS |
| 地图 SDK | **Leaflet + 天地图（Tianditu）** | 国家测绘局官方，国内覆盖优于 OSM，免费申请 Key，个人日调用量 10 万次 |
| 后端运行时 | Bun + Hono | 高性能 JS 运行时，轻量 Web 框架 |
| 数据库 | SQLite (better-sqlite3) | 零配置，单文件 |
| 文件存储 | 本地磁盘 `uploads/` | 图片直存本地 |
| 运行方式 | `bun run dev` / `bun run start` | 本地进程，无需 Docker |

### 5.1 天地图 vs 其他国内服务 对比

| 维度 | 天地图（推荐） | OpenStreetMap | 高德 JS API |
|---|---|---|---|
| API Key | 需免费申请 | **不需要** | 需申请 |
| 费用 | 个人免费（日 10 万次） | 完全免费 | 免费额度有限 |
| 国内道路/建筑细节 | **优秀**（官方测绘数据） | 一般（城市覆盖尚可） | 优秀 |
| 逆地理编码 | 需额外申请 | 无（本产品不需要） | 内置 |
| 与 Leaflet 集成 | 直接拼接瓦片 URL | 原生支持 | 需专用 SDK |
| 瓦片加载速度（国内） | **快**（国内 CDN） | 慢/不稳定 | 快 |

**决策**：使用 Leaflet 作为地图框架 + 天地图瓦片作为底图。Leaflet 负责交互层（标记、弹窗、点击事件），天地图提供底图瓦片。在 `leaflet` 基础上**无需额外 SDK**，直接使用天地图 WMTS 瓦片 URL 即可。

```js
// 天地图瓦片 URL 格式（需替换 YOUR_KEY）
L.tileLayer('https://t{s}.tianditu.gov.cn/vec_w/wmts?SERVICE=WMTS&REQUEST=GetTile&VERSION=1.0.0&LAYER=vec&STYLE=default&TILEMATRIXSET=w&FORMAT=tiles&TILEMATRIX={z}&TILEROW={y}&TILECOL={x}&tk=YOUR_KEY', {
  subdomains: ['0', '1', '2', '3', '4', '5', '6', '7'],
  maxZoom: 18,
})
```

**申请 Key**：访问 [天地图注册页](https://console.tianditu.gov.cn/) 注册并创建应用，选择"浏览器端"类型，获取 `tk` 参数值。

## 6. 非功能性需求

### 6.1 性能

| 指标 | 目标 |
|---|---|
| 首页首屏加载 | 本地运行，≤ 1s |
| 图片上传 | 前端压缩至 ≤ 1MB，单张 ≤ 3s |
| 地图渲染 | 标记 ≤ 200 个时流畅 |
| API 响应 | 本地 SQLite，P95 ≤ 50ms |

### 6.2 移动端适配

- 视口：`width=device-width, initial-scale=1, maximum-scale=1`
- 最低兼容：iOS Safari 14+ / Android Chrome 90+
- 触摸交互：地图单指拖动、双指缩放；列表原生滚动
- 图片上传：`<input capture>` 调起相机 + 相册选择

### 6.3 可靠性

- 图片上传失败 → 不创建记录，提示重试
- 数据库写入失败 → 返回 500，前端展示错误提示
- 天地图瓦片加载失败 → 显示提示并回退到 OSM 底图
- 编辑时图片未更换 → 保留原图路径不变

### 6.4 安全（本地运行，基础防护）

- 上传文件类型校验（仅允许 image/jpeg, image/png, image/webp）
- 上传文件大小服务端二次校验 ≤ 1MB
- 请求参数 Zod 校验，参数化查询防 SQL 注入
- 删除记录时同步删除关联图片文件

## 7. 前端设计参考

### 7.1 风格方向

- **极简实用**：无装饰元素，信息层级清晰
- **以图为主**：美食照片是最重要的视觉元素
- **移动优先**：所有交互基于拇指操作区设计

### 7.2 配色参考

| 用途 | 色值 | 说明 |
|---|---|---|
| 主色 | `#E86B35` | 暖橙色，食欲感 |
| 辅助色 | `#2D2D2D` | 深灰，文字主色 |
| 背景 | `#FAFAFA` | 浅灰底 |
| 卡片 | `#FFFFFF` | 白色卡片 |
| 分类标签 | 低饱和度彩色 | 柔和区分品类 |
| 危险操作 | `#EF4444` | 删除按钮 |

### 7.3 列表卡片样式

```
┌──────────────────────────────┐
│ ┌─────┐                      │
│ │     │  菜名                 │
│ │ 图  │  店名 · 分类标签      │
│ │     │  📍 地址文本          │
│ └─────┘                      │
└──────────────────────────────┘
```
- 图片：80×80 圆角缩略图，左侧固定
- 菜名：16px 加粗，单行截断
- 店名 + 分类：14px，灰色 + 彩色标签
- 地址：12px，灰色，单行截断

### 7.4 详情页操作栏

```
┌──────────────────────────────┐
│     [ ✏️ 编辑 ]  [ 🗑️ 删除 ]  │
└──────────────────────────────┘
```
- 编辑：主色按钮，跳转编辑页
- 删除：红色文字按钮，点击弹出确认框

### 7.5 地图标记样式

- 默认：Leaflet 默认标记或自定义橙色圆点（内联 SVG，无需外部图标文件）
- 点击态：弹出气泡显示菜名
- 聚合：20+ 个相近点时合并展示数量（Leaflet.markercluster 插件，可选）
- 底图：天地图矢量瓦片（vec_w），可叠加文字注记层（cva_w）
